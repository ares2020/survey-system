## 一、项目概述

### 1.1 这是什么

"共青团服务和促进大学生就业行动"周报填报系统，部署在 Render 上，供上海 70 所高校定期填报就业服务工作数据。

### 1.2 两个端

| 端 | 地址 | 用途 |
|:---|:---|:---|
| **填报端** | `/survey.html` | 70所高校填写周报（3页表单） |
| **管理端** | `/admin.html` | 管理员查看统计、导出Excel、管理记录 |

### 1.3 部署环境

- **平台**: Render.com (Free tier)
- **运行时**: Node.js 20
- **数据库**: JSON文件（默认）/ MongoDB Atlas（可选，V19新增）
- **数据持久化**: Render Disk 挂载 `/opt/render/project/data`
- **当前网址**: `https://survey-system-XXXX.onrender.com`（具体地址问用户）
- **管理员**: `admin` / `admin123`

## 四、关键代码说明（继续开发必读）

### 4.1 前端嵌套字段 → 后端扁平字段映射

这是**最容易出bug的地方**。前端用嵌套对象（`module1.lectureNew`），后端用扁平字段（`q8_weekly_lectures`）。

**映射函数**: `server/routes.js` 的 `mapSubmission()`（~189-362行）

```javascript
// 前端发来的结构
body.module1.lectureNew      → 后端: q8_weekly_lectures
body.module1.lectureCover    → 后端: q9_weekly_reach
body.module2.recruitNew      → 后端: q16_weekly_recruit
// ... 共50+个字段映射

// 子表映射
body.researchItems[]         → 后端: research_items[]
body.publicityItems[]        → 后端: publicity_items[]
body.plannedActivities[]     → 后端: planned_activities[]
body.reports[]               → 后端: completed_activities[]
```

**⚠️ 修改时务必保持前后端字段映射一致！**

### 4.2 数据库双模式

`server/db.js` 设计了 **MongoDB + JSON fallback** 双模式：

```javascript
// 判断当前模式
if (useMongo()) {
  // MongoDB模式：用 mongoose API 读写
  await Submission.findOneAndUpdate(...)
} else {
  // JSON模式：读写内存data对象 + saveData()落盘
  db.submissions.push(...)
  saveData() // 写入 data.json
}
```

**关键**: 所有路由代码通过 `getDb()` 获取内存数据对象，不直接操作 mongoose。这样两种模式下路由代码一样。

### 4.3 审计日志记录方式

```javascript
// 异步记录，不阻塞主流程
logAudit({
  action: 'CREATE',           // CREATE/UPDATE/DELETE/LOGIN/EXPORT/PASSWORD_CHANGE/RESTORE
  user: 'admin',              // 操作用户
  target_id: 123,             // 影响的submission id
  target_type: 'submission',
  details: { ... },           // 额外信息
  ip: '1.2.3.4'              // 客户端IP
}).catch(() => {})            // 忽略审计日志失败，不影响主业务
```

JSON模式下审计日志静默跳过（因为AuditLog是MongoDB模型）。

### 4.4 自动保存数据结构

`public/survey.html` 的 localStorage key: `survey_autosave`

```javascript
{
  // 第1页
  universityInput, university, otherUniversity,

  periodStart, periodEnd,
  // 第2页
  m1_lectureNew, m1_lectureCover, ...,

```

## 五、已知问题与注意事项

### 5.1 当前已知限制

| # | 问题 | 严重程度 | 说明 |
|:---|:---|:---:|:---|
| 1 | JSON模式下审计日志不可用 | 低 | 只有MongoDB模式才有审计日志持久化 |
| 2 | 批量导入前端需SheetJS CDN | 低 | 如果CDN不可用，导入功能失效 |
| 3 | 免费版Render 15分钟休眠 | 中 | 首次访问需等待冷启动（~30秒） |
| 4 | 无数据备份机制 | 中 | JSON模式下只有单份data.json，无自动备份 |
| 5 | 管理员只有一个 | 低 | 只支持单管理员账号 |
| 6 | 无邮件通知功能 | 低 | 未提交高校无自动提醒 |

### 5.2 历史踩坑记录（非常重要）

这些坑之前都踩过，**不要再踩**：

1. **`</script>` 缺少 `>`** — 会导致整个JS不执行。删除代码时务必检查HTML标签闭合。
2. **`m2` 变量未定义** — `validateBody()` 中使用前必须先 `const m2 = body.module2 || {}`
3. **前端字段名与后端不匹配** — `mapSubmission()` 中任何字段映射错误都会导致数据为0/空
4. **Render Disk路径配置** — Mount Path 必须是 `/opt/render/project/data`，不能是 `server/`
5. **CSP头拦截下载** — 已移除CSP，不要再加回去
6. **负数校验** — 数字字段需校验 `n < 0`，之前负数会导致覆盖率101%
7. **JWT环境变量** — V18之前硬编码，V19改为环境变量，不设置会登录失败
