## 场景一：全新部署（从零开始）

### 第一步：解压项目文件

```bash
# 在你的工作目录下创建项目文件夹
mkdir survey-system
cd survey-system

# 解压 V19 包（将 survey-system-v19.tar.gz 放到当前目录后执行）
tar -xzf survey-system-v19.tar.gz

# 查看解压后的文件结构
ls -la
# 应该看到：index.js  render.yaml  server/  public/
```

### 第二步：安装依赖

```bash
cd server
npm install

# 安装完成后确认 mongoose 已安装
ls node_modules | grep mongoose
# 输出：mongoose

cd ..  # 回到项目根目录
```

### 第三步：初始化本地数据文件（JSON 模式）

```bash
# 创建数据目录
mkdir -p data

# 初始化空数据文件
echo '{"submissions":[],"admins":[]}' > data/data.json

# 设置数据目录环境变量（当前终端会话有效）
export DATA_DIR="$(pwd)/data"
```

### 第四步：本地测试运行

```bash
# 在项目根目录执行
node index.js

# 期望输出：
# MONGODB_URI not set, using JSON file database
# Created data directory: ...
# Database initialized, nextId: 1
# Data file: .../data/data.json
# Submissions count: 0
# Server running on port 3001
```

打开浏览器访问：
- 填报页：`http://localhost:3001/survey.html`
- 管理后台：`http://localhost:3001/admin.html`
- 管理员账号：`admin` / `admin123`

按 `Ctrl+C` 停止服务。

### 第五步：创建 Git 仓库并推送到 GitHub

```bash
# 初始化 Git 仓库
git init

# 添加所有文件
git add -A

# 提交
git commit -m "V19 init: MongoDB + auto-save + recycle bin + audit log + batch import"

# 连接到 GitHub 仓库（先要在 GitHub 上新建空仓库，名为 survey-system）
git remote add origin https://github.com/你的用户名/survey-system.git

# 推送
git branch -M main
git push -u origin main
```

### 第六步：在 Render 上部署

1. 打开 https://dashboard.render.com 并登录
2. 点击 **New +** → **Web Service**
3. 选择 **Build and deploy from a Git repository**
4. 找到并选择你的 `survey-system` 仓库，点击 **Connect**
5. 填写以下配置：

| 字段 | 值 |
|:---|:---|
| **Name** | `survey-system` |
| **Region** | `Singapore` |
| **Branch** | `main` |
| **Runtime** | `Node` |
| **Build Command** | `cd server && npm install` |
| **Start Command** | `cd server && node index.js` |
| **Plan** | `Free` |

6. 在 **Environment Variables** 区域点击 **Add Environment Variable**，添加：

| Key | Value |
|:---|:---|
| `DATA_DIR` | `/opt/render/project/data` |
| `NODE_ENV` | `production` |

7. 在页面下方找到 **Disks** 区域，点击 **Add Disk**：

| 字段 | 值 |
|:---|:---|
| **Name** | `data` |
| **Mount Path** | `/opt/render/project/data` |
| **Size** | `1 GB` |

> ⚠️ **Mount Path 必须是 `/opt/render/project/data`，不能是 `server/` 或其他路径！** 否则 `npm install` 会覆盖数据文件。

8. 点击页面底部 **Create Web Service**
9. 等待 2-3 分钟，看到 **Your service is live** 绿色提示

### 第七步：验证部署

访问你的 Render 地址：`https://survey-system-XXXX.onrender.com`

- 填报页：`/survey.html`
- 管理后台：`/admin.html`
- 管理员：`admin` / `admin123`

## 场景二：V18 升级到 V19

> V18 的特征：有 `server/data.json` 文件，无 `server/models.js`，使用 `JWT_SECRET = 'survey-system-secret-key-2026'` 硬编码。

### 第一步：备份现有数据和代码

```bash
# 进入你的项目目录
cd survey-system

# 备份整个项目（最重要的一步！）
cd ..
cp -r survey-system survey-system-backup-v18-$(date +%Y%m%d-%H%M%S)

# 确认备份成功
ls -la survey-system-backup*
```

### 第二步：下载 V19 包

```bash
# 回到项目目录
cd survey-system

# 下载 V19 包到临时位置
# （将 survey-system-v19.tar.gz 放到 survey-system 同级目录后执行）
cp ../survey-system-v19.tar.gz /tmp/
```

### 第三步：保留数据文件

```bash
# V18 的数据文件位置（根据你的配置可能不同，以下列出常见位置）
# 先找到你的数据文件
find . -name "data.json" -type f 2>/dev/null

# 常见位置1：server/data.json
# 常见位置2：data/data.json（如果你有 DATA_DIR 环境变量指向这里）
# 常见位置3：/opt/render/project/data/data.json（Render 生产环境）

# 备份数据文件（假设在 server/data.json）
cp server/data.json /tmp/data.json.backup.$(date +%Y%m%d)

# 如果数据在 DATA_DIR 指定的目录，也备份
if [ -n "$DATA_DIR" ] && [ -f "$DATA_DIR/data.json" ]; then
    cp "$DATA_DIR/data.json" /tmp/data.json.backup.$(date +%Y%m%d).datadir
fi

echo "数据已备份到 /tmp/"
ls -la /tmp/data.json.backup.*
```

### 第四步：清理旧代码（保留数据）

```bash
cd survey-system

# 删除旧的服务端代码（保留 data.json！）
rm -rf server/*.js server/package.json

# 删除旧的前端代码
rm -rf public/*.html

# 删除根目录文件（后面用 V19 的替换）
rm -f index.js render.yaml

# 确认 data.json 还在
ls server/data.json 2>/dev/null || echo "注意：server/data.json 不存在，可能在 DATA_DIR 目录"
```

### 第五步：解压 V19 代码

```bash
# 解压 V19 包到当前目录
tar -xzf /tmp/survey-system-v19.tar.gz

# 确认文件结构
ls -la
ls server/
ls public/
```

### 第六步：恢复数据文件

```bash
cd survey-system

# 情况1：如果你之前用 server/data.json（V18默认）
if [ -f /tmp/data.json.backup.*.server ]; then
    cp /tmp/data.json.backup.*.server server/data.json
fi

# 情况2：如果你配置了 DATA_DIR 环境变量
if [ -n "$DATA_DIR" ] && [ -f /tmp/data.json.backup.*.datadir ]; then
    cp /tmp/data.json.backup.*.datadir "$DATA_DIR/data.json"
fi

# 情况3：如果没有 DATA_DIR，数据在 server/data.json（最常见）
# 已经在 server/ 目录下，V19 的 db.js 会默认读取当前工作目录或 DATA_DIR

# 确保数据文件存在且可读
ls -la server/data.json 2>/dev/null || echo "数据文件将在首次运行时自动创建"
```

### 第七步：安装新依赖（mongoose）

```bash
cd server

# 清理旧依赖（可选但推荐，避免冲突）
rm -rf node_modules package-lock.json

# 重新安装所有依赖（包含新增的 mongoose）
npm install

# 确认 mongoose 已安装
ls node_modules/mongoose/package.json

cd ..
```

### 第八步：本地测试

```bash
# 在项目根目录
export DATA_DIR="$(pwd)/server"
node index.js

# 期望输出：
# MONGODB_URI not set, using JSON file database
# Database initialized, nextId: ...
# Data file: .../server/data.json
# Submissions count: ...（显示你原来的数据条数）
# Server running on port 3001
```

测试内容：
1. 打开 `http://localhost:3001/admin.html`，用 admin/admin123 登录
2. 确认之前的提交记录都在
3. 测试删除一条记录 → 应该变为软删除（刷新后记录消失）
4. 查看回收站（新侧边栏菜单）→ 被删除的记录应该在回收站里
5. 测试恢复功能
6. 测试密码修改功能

按 `Ctrl+C` 停止。

### 第九步：提交并推送到 Render

```bash
# 添加所有更改
git add -A

# 提交
git commit -m "Upgrade to V19: MongoDB + auto-save + recycle bin + audit log + batch import"

# 推送
git push origin main
```

Render 会自动检测到推送并重新部署。

### 第十步：Render 环境变量更新

1. 打开 https://dashboard.render.com → 你的 Web Service
2. 点击 **Environment** 标签
3. 确认已有：
   - `DATA_DIR` = `/opt/render/project/data`
   - `NODE_ENV` = `production`
4. 新增环境变量（可选，不设置则自动使用 JSON 数据库）：
   - `JWT_SECRET` → 点击 **Generate** 按钮生成随机值（如果之前没有设置的话）
   - `MONGODB_URI` → 留空（如果暂时不用 MongoDB）
5. 点击 **Save Changes**
6. Render 会自动重新部署

> ⚠️ **重要**：如果之前 V18 的 JWT_SECRET 是硬编码的，升级后所有已登录的管理员需要重新登录（这是正常的安全行为）。

## 场景四：MongoDB Atlas 配置（可选）

> V19 不配置 MongoDB 也能正常工作（自动使用 JSON 文件数据库）。配置 MongoDB 可以获得更好的性能和数据可靠性。

### 第一步：注册 MongoDB Atlas（免费）

1. 打开 https://www.mongodb.com/atlas
2. 点击 **Try Free** 注册账号
3. 创建新的 Cluster：
   - 选择 **M0（免费）** 级别
   - 选择区域：`AWS / Singapore (ap-southeast-1)`
   - Cluster 名称：`survey-cluster`
4. 点击 **Create Deployment**
5. 等待集群创建完成（约 2-3 分钟）

### 第二步：创建数据库用户

1. 在左侧菜单点击 **Database Access**
2. 点击 **Add New Database User**
3. 填写：
   - **Authentication Method**: Password
   - **Username**: `survey_user`
   - **Password**: 点击 **Autogenerate Secure Password**，复制生成的密码
   - **Database User Privileges**: Read and write to any database
4. 点击 **Add User**

### 第三步：允许网络访问

1. 在左侧菜单点击 **Network Access**
2. 点击 **Add IP Address**
3. 选择 **Allow Access from Anywhere**（0.0.0.0/0）
   - 或者只添加 Render 的 IP 段（可选）
4. 点击 **Confirm**

### 第四步：获取连接字符串

1. 在左侧菜单点击 **Database** → 点击 **Connect**
2. 选择 **Drivers**
3. 选择 **Node.js**，版本 **5.5 或更高**
4. 复制连接字符串，格式类似：
   ```
   mongodb+srv://survey_user:<db_password>@survey-cluster.xxxxx.mongodb.net/survey?retryWrites=true&w=majority&appName=survey-cluster
   ```
5. 将 `<db_password>` 替换为你刚才创建的数据库用户密码

### 第五步：配置到 Render

1. 打开 https://dashboard.render.com → 你的 Web Service → **Environment** 标签
2. 点击 **Add Environment Variable**
3. 添加：
   - Key: `MONGODB_URI`
   - Value: 上面的连接字符串（替换密码后）
4. 点击 **Save Changes**
5. Render 会自动重新部署

### 第六步：验证 MongoDB 连接

查看 Render 部署日志（**Logs** 标签），应该看到：

```
MongoDB connected successfully
Database initialized, nextId: ...
Submissions count: ...
Server running on port 10000
```

而不是之前的 `MONGODB_URI not set, using JSON file database`。

### 第七步：数据迁移（首次启用 MongoDB 时）

V19 会在首次连接 MongoDB 时自动将 JSON 文件中的数据迁移到 MongoDB。查看日志确认：

```
MongoDB is empty, migrating N submissions and M admins from JSON
Migration complete: N submissions, M admins
```

## 常见问题排查

### 问题1：启动报错 `Cannot find module 'mongoose'`

```bash
cd server
npm install
# 如果还是失败
cd server && rm -rf node_modules && npm install
cd ..
node index.js
```

### 问题2：启动报错 `MongooseServerSelectionError`

说明 MongoDB 连接失败。检查：

```bash
# 确认 MONGODB_URI 是否正确
echo $MONGODB_URI

# 如果留空或未设置，系统会自动使用 JSON fallback（这是正常的）
# 如果想禁用 MongoDB 尝试，确保环境变量不存在：
unset MONGODB_URI
```

### 问题3：部署后数据丢失

检查 Render 环境变量：

```
DATA_DIR=/opt/render/project/data
```

检查 Disk 配置：

```
Mount Path=/opt/render/project/data
```

两者必须完全匹配。

### 问题4：管理员登录失败（升级后）

V19 的 JWT_SECRET 改为环境变量驱动。如果之前硬编码的密钥与新环境变量不同，已颁发的 Token 会失效。这是正常行为，重新登录即可。

如果完全无法登录：

```bash
# 停止服务，删除 data.json 重置管理员
rm server/data.json  # 或 $DATA_DIR/data.json
# 重新启动，系统会自动创建默认管理员 admin/admin123
node index.js
```

### 问题5：`EACCES: permission denied` 数据文件权限

```bash
chmod 755 data/
chmod 644 data/data.json
```

### 问题6：git push 被拒

```bash
git remote -v  # 确认 remote 地址正确
git pull origin main --rebase  # 先拉取远程更新
git push origin main
```

## 📞 回滚方案

如果 V19 部署后出现问题，快速回滚到 V18：

```bash
# 本地回滚
cd survey-system
git log --oneline -5  # 找到 V18 的 commit
git checkout <V18-commit-hash>  # 切换到 V18
# 或者
cd ..
rm -rf survey-system
cp -r survey-system-backup-v18-XXXXXXXX survey-system
cd survey-system/server && npm install && cd ..
node index.js
```

Render 回滚：
1. 在 Render Dashboard → 你的 Web Service
2. 点击 **Manual Deploy** → **Deploy latest commit**
3. 或者选择之前的部署记录点击 **Rollback**
